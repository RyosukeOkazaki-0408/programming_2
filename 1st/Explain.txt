private static void dfs(Room room, int id) {
    int[][] vis = room.getVis();
    int size = room.getSize();
    int x = -1, y = -1;

    for (int i = 0; i < size; i++) {
        for (int j = 0; j < size; j++) {
            if (vis[i][j] == 0) {
                x = i;
                y = j;
                break;
            }
        }
        if (x != -1) { break; }
    }

    if (x == -1) {
        System.out.println("---------------\n" + ++cnt + "個目（" + room.getSunpo() + "畳半）\n\n" + room);
        return;
    }

    Tatami yokoTatami = new Tatami(id, x, y, true);
    Tatami tateTatami = new Tatami(id, x, y, false);
    if(room.setTatami(tateTatami)) {
        dfs(room, id+1);
        room.removeTatami(tateTatami);
    }
    if(room.setTatami(yokoTatami)) {
        dfs(room, id+1);
        room.removeTatami(yokoTatami);
    }
}
→ privateなmethodである。
→ 左上を起点に探索をすすめ、畳が置かれていない（かつ炉がない）場所があれば畳を設置できるか縦と横それぞれで判定する。
→ もし置ければ（setTatami() == true）より深く探索を続け、置けなければ（false）探索を終了する。
→ 探索後はremoveTatami()でバックトラック処理を行っている。

}

• 考察
畳の縁が十字になってはいけないとのことだったため、炉と畳からなる十字は十字でないとみなして考えた。
処理を高速に行うため、畳を設置するごとに十字を探す処理を入れ（置いた畳の周辺のみ）、バックトラッキングを行なっている。
また、再帰に使われる関数や特定のクラスでしか用いないメソッドなどは、極力privateでカプセル化することを意識している。
引数が偶数（炉が置けない）であったり、1以下の値（畳が置けなかったり、サイズとして不正な値）であった場合は、エラーメッセージが出るようにしている。

表示するのをアルファベットではなく数値にしてformatすればもっと大きな引数でも実行できると考えたが、課題の指示に従ってアルファベットのままにした。

全体的に構造が複雑で、特に十字を見つける処理がとても煩雑になってしまったため、かなり改善の余地があると思う（方法は浮かばないが）。
どう考えてもこの解法は想定解ではないため、問題の想定解についての解説をしてほしいなと思った。
